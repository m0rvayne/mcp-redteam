01Executive Summary

We audited 11 MCP servers (287 tools) across four categories: health, architecture, completeness, and security. Each server was assigned a dedicated attack agent that read its source code and systematically probed every tool.

We found 5 critical vulnerabilities. We extracted live Trello API credentials from a plaintext .env file. We read an Instagram session cookie granting full account access. We found Google OAuth tokens stored world-readable on disk across 2 servers. We discovered an AppleScript injection in MindNode that allows arbitrary code execution through unescaped clipboard content.

Beyond security, we found blocking synchronous calls in 6 servers that freeze the event loop, unbounded downloads that can OOM-crash 2 servers, and no .gitignore in 8 out of 11 projects — meaning any accidental git push leaks credentials.

Three servers demonstrated strong security posture: meeting-transcript (exemplary path traversal protection), youtube (proper URL validation), and fathom (bounded pagination, Pydantic validation, rate-limit awareness).

03Audit Results
mindnode Critical
AppleScript injection via setClipboard — newlines not escaped, allows arbitrary code executionSecurity
What we found
setClipboard() escapes \ and " but NOT newlines. A payload like foo\n" & (do shell script "id") & " breaks out of the AppleScript string and executes shell commands. Any tool that sets clipboard content can execute arbitrary commands as the current user.
execSync shell usage in activateMindNode — dangerous patternSecurity
What we found
Uses execSync with shell invocation for activating MindNode, introducing potential command injection surface.
All calls synchronous (execFileSync) — blocks event loopHealth
What we found
Every tool call uses synchronous execFileSync to call Swift bridge binaries. Blocks for up to 30 seconds per operation. No concurrent processing possible.
Swift bridge manual JSON serialization — incomplete escapingArchitecture
What we found
JSON is built by string concatenation. The jsonEscape function handles \, ", \n, \r, \t but not other control characters (NUL, form feed). Produces invalid JSON on edge-case input.
No .gitignore — node_modules exposedCompleteness
What we found
No .gitignore present. node_modules/ and build artifacts will be committed on any git init && git add .
  • execFileSync with array args (no command injection)
  • Timeouts on all exec calls
  • Error containment

cookies.txt with live sessionid stored plaintext, permissions 644Security

Contains a live sessionid, csrftoken, and ds_user_id. File permissions 644 — readable by any process. Grants full Instagram account impersonation.

Unbounded file download into memory — OOM riskHealth
What we found
download_bytes() loads entire files into BytesIO. No size check before download. A large file exhausts RAM and crashes the process.
upload_file allows exfiltrating ~/.ssh, ~/.aws from home dirSecurity
What we found
upload_file restricts local_path to be under $HOME — but that includes ~/.ssh/id_rsa, ~/.aws/credentials, ~/.gnupg/. A prompt injection can instruct the LLM to upload private keys to Drive.
  • Path traversal defended in download_file (is_relative_to)
  • No SSRF vectors
  • Signal handling
